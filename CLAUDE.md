# CLAUDE.md

このファイルは、本リポジトリで作業する Claude Code / 開発者向けのガイドです。
詳細な仕様は [SPEC.md](SPEC.md)、実装計画は [TASKS.md](TASKS.md) を参照してください。

## プロジェクト概要

楽譜のスクリーンショットを読み込み、各音符が何の音か（C4, D4 …）と、和音から推定したコード名を、**画像に重ねて**表示するWebアプリ。主用途は **対面 / 画面共有でのライブ指導**（人に教えるため）。

想定する入力は、YouTube の「楽譜ダウンロード」系動画のスクショなど、**きれいにレンダリングされた大譜表（トレブル＋バス）の五線譜**の静止画一枚。写真・手書き・動画URLは対象外（将来検討）。

## 実行方法

- ビルド不要。`index.html` をPCのWebブラウザで開くだけ。
- インストール不要・オフラインで動作すること。

## 譲れない原則（設計制約）

これらは要件から導かれた強い制約であり、逸脱する場合は必ずユーザーに確認すること。

1. **単一HTML・オフライン・ローカル完結**：`index.html` をローカルで開けば動く。ビルドステップや起動手順を増やさない。
2. **無料・送信なし・課金しない**：画像を外部に送信しない。ブラウザ内で処理を完結する。外部API（クラウドOMR/画像認識/LLM等）を、ユーザーの事前承認なしに導入・課金しない。依存ライブラリを使う場合はローカルに vendor（同梱）し、CDN 実行時取得に依存しない。
3. **理論エンジンが信頼できる中核／CVはベストエフォート**：音名判定（五線位置→C4）とコード判定（音集合→コード名）は純粋なロジックで正確に作る。画像認識（CV）は不確実な補助であり、**人による確認・修正を必須**とする。誤った音名を出さないことが指導用途では最優先。
4. **半自動を前提にした修正UI**：自動認識の結果は常にクリックで訂正できる。CVが弱くても、手動アノテーションだけでアプリが実用になる状態を保つ。

## アーキテクチャの芯

```
[画像読込] → [五線/音部記号/調号の把握] → [符頭検出(CV, 補助)]
        ↘ 手動アノテーション/修正(必須) ↙
                     ↓
         [音名エンジン: 五線位置 → C4 表記]
                     ↓
         [コードエンジン: 拍/小節で音をまとめ → コード名]
                     ↓
         [Canvas 上に音名・コードをラベル重ね（密度切替え）]
```

- **音名エンジン**と**コードエンジン**は画像から独立した純ロジック。単体でテスト可能に保つ。
- CV（画像認識）層は「プリフィルする補助」であり、出力の正しさを保証しない。

## コーディング方針

- 依存は最小限。まずバニラ JS + Canvas を第一候補にする。ライブラリ導入時はローカル同梱し、オフライン動作を壊さない。
- 音楽理論のロジック（音名・コード）はUIから分離し、副作用のない関数として実装する。
- コメントは「なぜ」が非自明な箇所のみ。表記規則やオクターブ定義など、間違えやすい前提には根拠を残す。

## 主要な確定事項（詳細は SPEC.md）

- 音名表記：英米式 C4/D4。**真ん中のC = C4**（科学的音高表記, MIDI 60）。
- 黒鍵の表記：**調号・臨時記号に従う**（例：D長調なら F#）。→ 音部記号・調号の把握が必要。
- コードのまとめ方：**拍/小節でまとめて**和声推定（アルペジオ対応）。テンション（9th/sus4/add9等）や分数/オンベースも可能な範囲で。解釈が割れる場合は候補を提示。
- 音の再生：**不要**。
- ラベル密度：初期状態は**切替えで制御**（全音表示／コードのみ／選択した音のみ）。
