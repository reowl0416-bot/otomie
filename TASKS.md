## Phase 1 — 音楽理論エンジン（純ロジック・UI非依存）✅ done
実装: `engine.js` / テスト: `engine.test.js`（32 assertions passing）
> テスト実行: `node engine.test.js`（Node 未導入時は `cat engine.js engine.test.js | osascript -l JavaScript`）

音名エンジン
- [x] データモデル定義：音部記号 / 調号 / 臨時記号 / 五線上の垂直位置（段位置）。
- [x] 五線位置 × 音部記号 → 音高（ダイアトニック段位）算出。加線対応。
- [x] 調号 + 臨時記号 → 変化記号の反映（同小節継続・小節線リセット）。
- [x] 音高 → 英米式表記 `C4`（真ん中のC = C4 / MIDI 60）。異名同音は**調号・臨時記号に従う**。
- [x] 単体テスト：代表調・加線・臨時記号のケース。

コードエンジン
- [x] 音集合 → コード名推定（メジャー/マイナー/7th、テンション 9th/sus4/add9 等）。
- [x] 最低音がルートでない場合の分数/オンベース表記 `C/E`。
- [x] 複数解釈時の**候補リスト**出力（ルート=最低音の基本形を優先）。
- [x] 単体テスト：三和音〜テンション、転回形、分数コード。

**完了条件**：音の集合を与えると、音名列とコード名（＋候補）が正しく返る。画像なしで検証可能。 ✅

## Phase 3 — 画像認識(CV) 自動プリフィル（ブラウザ内・補助）
実装: `cv.js`（純画像処理）/ `index.html`（「自動認識」ボタンでプリフィル）
- [x] グレースケール化・二値化（Otsu）。
- [x] 水平投影で**五線検出**（各段の5本線・段位置・線間隔）。合成データで検証済。
- [x] 段 → 小節線の推定。縦投影で小節線検出（上下端の両方にインクがある列＝小節線。茎は片側で途切れるため除外）。小節区間を構築し、指導向けの**小節送り（レッスン）UI**（1小節ずつ音名・コード・解説を表示、←/→で移動）に接続。※拍の細分は未着手。
- [x] **符頭検出**（積分画像＋矩形フィル率／塗り・白抜き両対応）→ 五線位置へ写像。※ベストエフォート。
- [~] 音部記号・調号の自動把握：音部記号は段の上下位置で自動割当（上=ト音）。**調号は手動選択を採用**（自動検出は未実装）。失敗時は手動フォールバック可。
- [x] 認識結果を Phase 2 のアノテーションへ**プリフィル**（すべて修正可能なまま）。

**完了条件**：一定フォーマットのスクショで、多くの音符・調号が自動で埋まり、人が仕上げれば正しくなる。
> 実画像での精度検証はブラウザ実機で行う（`getImageData` が必要なため）。閾値は `cv.js` の定数で調整可能。
